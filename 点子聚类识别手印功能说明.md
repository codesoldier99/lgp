# 点子聚类识别手印功能说明

## 功能概述

本次更新添加了点子聚类分析功能，可以自动识别聚集在一起的点子（麻点）并将其归类为"手印"瑕疵。这解决了设备会将手印上的每个点子单独检测出来的问题，现在系统可以智能地将满足条件的聚集点子合并为一个手印瑕疵。

## 识别条件

当一块区域内的点子满足以下**两个条件**时，将被识别为"手印"：

1. **距离条件**：各个点子之间的距离小于 **0.3mm**
2. **数量条件**：聚类的点子数量大于 **20个**

## 技术实现

### 1. 核心算法类

新增了 `SpotClusteringHelper` 类，位于：
- 文件路径：`IndustryDemo\Controllerui\SpotClusteringHelper.cs`

该类实现了改进的 DBSCAN（基于密度的空间聚类）算法，主要功能包括：

- **点子聚类**：自动识别距离相近的点子群
- **手印识别**：将满足条件的聚类标记为手印
- **坐标计算**：计算每个聚类的中心坐标作为手印位置
- **参数可配置**：支持自定义距离阈值和最小点数

### 2. 可视化组件

新增了手印可视化控件，使用橙色圆圈标识：
- `IndustryDemo\Controllerui\fingerprint.cs`
- `IndustryDemo\Controllerui\fingerprint.Designer.cs`
- `IndustryDemo\Controllerui\fingerprint.resx`

在虚拟盘上，手印会显示为：
- 橙色圆形边框（直径10像素）
- 中心有橙色填充点
- 与普通点子（灰色）明显区分

### 3. 检测流程集成

修改了点光源检测算法（`DetectionUnderSpotLight.cs`）：

```csharp
// 检测到点子后，自动进行聚类分析
SpotClusteringHelper clusteringHelper = new SpotClusteringHelper(
    distanceThresholdMm: 0.3,  // 距离阈值0.3mm
    minClusterSize: 20         // 最小聚类点数20个
);

List<string[]> fingerprintDefects;  // 识别出的手印
List<string[]> remainingSpots;      // 剩余的独立点子

clusteringHelper.ProcessSpotClustering(spotList, 
    out fingerprintDefects, 
    out remainingSpots);
```

### 4. 数据存储

- 手印瑕疵类型名称：`"手印"`
- 数据库存储：与其他瑕疵类型相同的表结构
- 坐标存储：使用聚类中心的行列坐标

## 使用方法

### 1. 自动模式（推荐）

功能已自动集成到现有检测流程中，无需额外配置：

1. 启动检测程序
2. 点击"开始检测"
3. 系统会自动：
   - 检测所有点子
   - 执行聚类分析
   - 识别手印并分类
   - 在虚拟盘上显示结果

### 2. 参数调整（高级）

如需调整聚类参数，可修改 `DetectionUnderSpotLight.cs` 中的参数：

```csharp
SpotClusteringHelper clusteringHelper = new SpotClusteringHelper(
    distanceThresholdMm: 0.3,  // 修改此值调整距离阈值（单位：毫米）
    minClusterSize: 20         // 修改此值调整最小点数要求
);
```

**参数说明**：
- `distanceThresholdMm`：点子之间的最大距离（毫米），超过此距离的点子不会被归为同一聚类
- `minClusterSize`：形成手印所需的最小点子数量
- `photoRatio`：像素到毫米的转换比例（默认使用 `Global.photoRadio`）

## 检测结果说明

### 1. 虚拟盘显示

- **手印**：橙色圆圈标识（显示在聚类中心位置）
- **独立点子**：灰色圆点（未能形成聚类的点子）
- **其他瑕疵**：按原有颜色显示

### 2. 数据库记录

每个识别出的手印会作为一条独立记录存储：
- `defectionType` = "手印"
- 坐标为聚类中心的位置
- 包含检测时间、二维码等信息

### 3. 报表输出

- 手印统计会显示在对应的文本框（`textEdit16`）
- 可按时间、类型查询手印记录
- 报表中会显示每个手印的位置和数量

## 算法优势

1. **智能识别**：自动区分手印和独立点子，避免过度报警
2. **准确定位**：使用聚类中心作为手印位置，定位更准确
3. **参数灵活**：可根据实际情况调整距离和数量阈值
4. **性能优化**：使用高效的DBSCAN算法，处理速度快

## 注意事项

1. **距离阈值设置**：
   - 设置过小：可能将一个手印拆分为多个聚类
   - 设置过大：可能将多个手印合并为一个聚类
   - 建议值：0.3mm（根据实际测试调整）

2. **最小点数设置**：
   - 设置过小：可能将普通污渍误判为手印
   - 设置过大：可能漏检不完整的手印
   - 建议值：20个（根据实际情况调整）

3. **照片分辨率**：
   - 系统使用 `Global.photoRadio` 进行像素到毫米的转换
   - 确保该值与实际相机分辨率匹配
   - 当前默认值：4.5 像素/毫米

4. **性能考虑**：
   - 聚类算法的时间复杂度为 O(n²)，n为点子数量
   - 对于大量点子（>1000个），可能需要几秒钟处理时间
   - 建议在独立线程中执行聚类分析（已实现）
